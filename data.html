Spectra

Data from all four instruments have been rewritten to a standard FITS format. The spectral data is stored in different HDUs, which are named by the below keywords. It should be noted that this standard data will have more NaNs than is typically expected.

Relative Orders Standardized to Echelle Order

All data have dimensions of [number of orders, number of pixels].

The orders have been re-organized such that the relative indexing of orders always maps to the same echelle order. In other words, the first order in every standardized FITS file corresponds to echelle order 161. Calling the Nth order of this standardized data will return an echelle order of a comparable wavelength regardless of the source instrument (e.g., H-alpha appears in the 68th entry of the standardized data for all instruments).

Data has been padded as necessary with NaNs. For instance, NEID orders start at echelle order 161 while EXPRES orders start at echelle order 160. Therefore, the first entry for every EXPRES file contains only NaNs. Of particular note is that the HARPS data does not contain echelle order 115 (relative order 46 in the standardized data format).

There is a simple function in the example code for reading spectra to generate a mask of orders that are all NaNs.
